Attack Tests

Attack 01

SQL Injection

Injects SQL syntax into the search query, manipulating the database to return all rows rather than matching products. A successful injection bypasses authentication and can expose every record in the database.

Test Payload
' OR '1'='1
GET /api/waf-test?attack=sqli
Attack 02

Cross-Site Scripting (XSS)

Injects a malicious script tag into a page field that gets rendered to other users. When executed in a victim's browser, the script can steal session cookies, hijack accounts, or silently redirect users to phishing pages.

Test Payload
<script>alert('XSS')</script>
GET /api/waf-test?attack=xss
Attack 03

Command Injection

Appends OS commands to an input that gets passed directly to a system shell on the origin server. A successful injection allows arbitrary command execution — exposing server files, credentials, and internal network information.

Test Payload
; cat /etc/passwd
GET /api/waf-test?attack=cmdi
Attack 04

HTTP Header Injection

Injects malicious HTTP headers — using CRLF sequences — to manipulate server responses, poison shared caches, or spoof trusted headers like X-Forwarded-For to bypass access controls. All visitors to a poisoned cache receive attacker-controlled content.

Test Payload
X-Forwarded-For: 127.0.0.1 X-Injected: malicious-payload
GET /api/waf-test?attack=header
